# Prompt Lab PRD 최종본
## Socratic Prompt Engineering Learning Platform for KDI School

---

## 1. 제품 개요

**제품명**: Prompt Lab (소크라테스식 프롬프트 엔지니어링 학습 플랫폼)

**비전**: KDI School 대학원생들이 일방적 강의가 아닌 소크라테스식 대화를 통해 스스로 프롬프트 엔지니어링의 핵심 원리를 체득하고, 정책 학습 및 리서치 상황에서 효과적인 프롬프트를 작성할 수 있도록 돕는 인터랙티브 웹 플랫폼

**핵심 가치**: "가르치지 않고 깨닫게 한다"

---

## 2. 문제 정의

### 현재 상황
- 프롬프트 엔지니어링 교육이 대부분 "좋은 프롬프트 예시"를 일방적으로 전달하는 방식
- 학생들이 왜 특정 프롬프트가 효과적인지 원리를 이해하지 못한 채 패턴만 암기
- 새로운 상황에서 적절한 프롬프트를 스스로 설계하지 못함
- 정책대학원생 특화 프롬프트 교육 부재

### 해결하고자 하는 문제
- 수동적 학습 → 능동적 발견 학습으로 전환
- 결과 비교를 통한 직관적 이해 촉진
- 정책 학습/리서치 맥락에 특화된 프롬프트 작성 역량 개발

---

## 3. 목표 사용자

### Primary: KDI School 정책대학원생 (석사/박사)

**특성**:
- 공공정책/개발정책 전공
- 국제학생 비율 높음 (다양한 국가 배경)
- AI 도구 활용 경험이 다양한 수준
- 한국어/영어 이중언어 사용

**주요 활동**:
- 정책 분석, 비교 연구, 사례 연구 수행
- 영어 논문/보고서 작성
- 캡스톤 프로젝트 수행
- 정책 브리핑, 메모, 제안서 작성

### Secondary: 교수자/강사
- 학생들의 학습 진도 모니터링
- 커리큘럼 설계 및 시나리오 관리

---

## 4. 핵심 기능

### 4.1 소크라테스식 대화 엔진 (Socratic Dialogue Engine)

**기능 설명**: AI 튜터가 프롬프트 엔지니어링 개념에 대해 직접 설명하지 않고, 유도 질문을 통해 학생이 스스로 깨닫도록 안내

**주요 질문 유형**:

| 질문 유형 | 목적 | 예시 |
|-----------|------|------|
| 탐색 질문 | 기존 지식 확인 | "AI에게 질문할 때 무엇이 결과에 영향을 준다고 생각하나요?" |
| 명료화 질문 | 생각 구체화 | "구체적으로 어떤 정보를 추가하면 더 나은 답변을 받을 수 있을까요?" |
| 가정 검증 질문 | 전제 점검 | "왜 그렇게 생각하나요? 다른 가능성은 없을까요?" |
| 결과 예측 질문 | 비판적 사고 | "이 프롬프트로 어떤 결과가 나올 것 같나요?" |

**대화 흐름**:
```
[시나리오 제시] → [유도 질문] → [학생 응답] → [심화 질문 or 다음 단계]
```

---

### 4.2 프롬프트 작성 실습 (Prompt Writing Lab)

**기능 설명**: 정책 학습/리서치 시나리오가 주어지면 학생이 직접 프롬프트를 작성

**시나리오 카테고리**:

| 카테고리 | 설명 | 예시 상황 |
|----------|------|-----------|
| 문헌 리뷰 | 학술 논문 검색/요약/정리 | 캡스톤 선행연구 조사 |
| 정책 비교 | 국가 간/정책 간 비교 분석 | Policy Analysis 과제 |
| 데이터 분석 | 통계 해석, 시각화 제안 | Quantitative Methods 과제 |
| 정책 문서 | 브리핑, 메모, 제안서 | 인턴십, 수업 과제 |
| 연구 설계 | 연구 질문 도출, 방법론 검토 | 논문 proposal |
| 이해관계자 분석 | Stakeholder mapping | 정책 과정 분석 |

---

### 4.3 비교 실험실 (Comparison Lab)

**기능 설명**: 학생 프롬프트 vs 개선된 프롬프트의 실제 AI 응답을 나란히 비교

**비교 항목 (자동 분석)**:
- 구체성 (Specificity)
- 맥락/배경 정보 (Context)
- Persona 설정 (Persona)
- 출력 형식 (Output Format)
- 제약 조건 (Constraints)
- 예시 포함 여부 (Examples)

---

### 4.4 성찰 저널 (Reflection Journal)

**기능 설명**: 비교 결과를 본 후 학생이 스스로 배운 점을 정리하도록 유도

**일반 성찰 질문**:
- "두 결과의 가장 큰 차이는 무엇이었나요?"
- "왜 그런 차이가 발생했다고 생각하나요?"
- "다음에 비슷한 상황에서 어떻게 프롬프트를 작성하시겠습니까?"

**테크닉 인식 유도 질문** (모듈별):

| 모듈 | 성찰 질문 |
|------|-----------|
| 1 | "복잡한 문제를 '단계별로' 분석해달라고 하면 왜 더 좋은 답이 나올까요?" |
| 2 | "예시를 하나 보여주는 것이 왜 효과적일까요? 몇 개의 예시가 적절할까요?" |
| 3 | "분석 순서를 미리 알려주는 것과 AI가 알아서 하게 두는 것의 차이는?" |
| 4 | "AI에게 전문가 정체성을 부여하면 응답이 어떻게 달라지나요?" |
| 5 | "지금까지 배운 방법들 중 가장 효과적이라고 느낀 것은 무엇인가요?" |

**저장 및 활용**:
- 개인 학습 포트폴리오로 누적
- 진행 상황 시각화 (성장 그래프)

---

### 4.5 학습 모듈 구성 (KDI School 맥락 기반)

#### 모듈 개요

| Module | 제목 | 내재된 테크닉 | 정책 맥락 |
|--------|------|---------------|-----------|
| 1 | 좋은 질문이 좋은 답을 만든다 | Chain of Thought | 정책 이슈 질문 |
| 2 | 문헌 리뷰 효과적으로 하기 | Few-shot + Output Structuring | 캡스톤 선행연구 |
| 3 | 정책 비교 분석 요청하기 | CoT + Structuring + Self-Consistency | 국가 간 정책 비교 |
| 4 | 데이터 해석 도움받기 | Persona + CoT + Few-shot | 통계/회귀분석 해석 |
| 5 | 정책 문서 작성 지원받기 | Persona + Few-shot + Constraints | 브리핑/메모/제안서 |

---

#### Module 1: 좋은 질문이 좋은 답을 만든다

| 항목 | 내용 |
|------|------|
| **학습 목표** | 프롬프트가 AI 응답 품질에 미치는 영향을 체험적으로 이해 |
| **내재된 테크닉** | Chain of Thought (Zero-shot CoT) |
| **정책 맥락** | 정책 이슈에 대한 질문 |

**소크라테스 질문 예시**:
- "교수님께 질문할 때와 AI에게 질문할 때, 어떤 점이 비슷하고 다를까요?"
- "같은 주제에 대해 모호하게 질문하면 어떤 답을 받을 것 같나요?"

**시나리오**:
```
당신은 "개발도상국의 보편적 기본소득(UBI) 정책"에 대해 
알아보려고 합니다. AI에게 어떻게 질문하시겠습니까?
```

**비교 실험**:

| 학생 프롬프트 (예상) | 개선된 프롬프트 |
|---------------------|-----------------|
| "UBI에 대해 알려줘" | "개발도상국에서 UBI 정책을 도입할 때 고려해야 할 사항을 **단계별로** 분석해주세요: (1) 먼저 재정적 실현가능성을 검토하고, (2) 기존 사회보장제도와의 관계를 분석하고, (3) 정치적 수용성을 평가하고, (4) 최종적으로 정책 제언을 도출해주세요." |

---

#### Module 2: 문헌 리뷰 효과적으로 하기

| 항목 | 내용 |
|------|------|
| **학습 목표** | 학술 문헌 검색, 요약, 정리에 효과적인 프롬프트 작성 |
| **내재된 테크닉** | Few-shot Learning + Output Structuring |
| **정책 맥락** | 학술 문헌 검색/요약 (캡스톤 선행연구) |

**소크라테스 질문 예시**:
- "문헌 리뷰할 때 가장 시간이 오래 걸리는 작업은 무엇인가요?"
- "AI가 논문을 요약해준다면, 어떤 정보가 포함되어야 유용할까요?"

**시나리오**:
```
캡스톤 프로젝트를 위해 "기후변화가 동남아시아 농업 정책에 
미친 영향"에 대한 문헌 리뷰를 시작하려 합니다.
AI에게 어떻게 도움을 요청하시겠습니까?
```

**비교 실험**:

| 학생 프롬프트 (예상) | 개선된 프롬프트 |
|---------------------|-----------------|
| "기후변화와 농업 정책 논문 요약해줘" | "아래 형식에 맞춰 논문들을 요약해주세요.<br><br>**[예시]**<br>• 저자/연도: Smith et al. (2020)<br>• 연구 질문: 탄소세가 농가 소득에 미치는 영향<br>• 방법론: 패널 데이터 분석 (2010-2019)<br>• 핵심 발견: 탄소세 1% 인상 시 농가 소득 0.3% 감소<br>• 정책 함의: 농가 보조금 병행 필요<br><br>이 형식으로 기후변화가 동남아시아 농업 정책에 미친 영향 관련 주요 연구 5개를 정리해주세요." |

#### Module 3: 정책 비교 분석 요청하기

| 항목 | 내용 |
|------|------|
| **학습 목표** | 국가 간/정책 간 비교 분석을 위한 구조화된 프롬프트 작성 |
| **내재된 테크닉** | Chain of Thought + Output Structuring + Self-Consistency |
| **정책 맥락** | 국가 간 정책 비교 (Policy Analysis 과제) |

**소크라테스 질문 예시**:
- "좋은 비교 분석에는 어떤 요소가 포함되어야 할까요?"
- "비교 기준 없이 '비교해줘'라고 하면 어떤 문제가 생길까요?"

**시나리오**:
```
Policy Analysis 수업 과제로 "한국과 독일의 재생에너지 정책"을 
비교해야 합니다. AI에게 어떻게 분석을 요청하시겠습니까?
```

**비교 실험**:

| 학생 프롬프트 (예상) | 개선된 프롬프트 |
|---------------------|-----------------|
| "한국이랑 독일 재생에너지 정책 비교해줘" | "한국과 독일의 재생에너지 정책을 비교 분석해주세요.<br><br>**분석 순서:**<br>1단계: 각 국가의 정책 배경과 목표를 먼저 정리<br>2단계: 정책 수단을 범주별로 비교 (규제, 인센티브, R&D)<br>3단계: 성과 지표로 효과성 평가<br>4단계: 각 정책의 강점과 약점 도출<br>5단계: 한국 관점에서 시사점 정리<br><br>마지막에 **반대 관점**도 검토해주세요: 독일 모델을 한국에 적용하기 어려운 이유는 무엇일까요?" |

#### Module 4: 데이터 해석 도움받기

| 항목 | 내용 |
|------|------|
| **학습 목표** | 통계 데이터 해석, 시각화 제안, 분석 방향 설정에 AI 활용 |
| **내재된 테크닉** | Persona 설정 + Chain of Thought + Few-shot Learning |
| **정책 맥락** | 회귀분석/통계 해석 (Quantitative Methods 과제) |

**소크라테스 질문 예시**:
- "데이터를 보여주기만 하면 AI가 당신의 연구 질문을 알 수 있을까요?"
- "AI에게 '전문가' 정체성을 부여하면 어떤 차이가 있을까요?"

**시나리오**:
```
World Bank 데이터로 "ODA가 수원국 교육 성과에 미친 영향"을 
분석하고 있습니다. 회귀분석 결과가 나왔는데 해석에 어려움을 
겪고 있습니다. AI에게 어떻게 도움을 요청하시겠습니까?
```

**비교 실험**:

| 학생 프롬프트 (예상) | 개선된 프롬프트 |
|---------------------|-----------------|
| "이 회귀분석 결과 해석해줘 [데이터]" | "**당신은 World Bank에서 10년간 개발도상국 교육 정책을 연구한 개발경제학자입니다. 데이터 기반 분석을 중시하며, 정책 입안자가 이해할 수 있는 언어로 설명합니다.**<br><br>아래 회귀분석 결과를 해석해주세요.<br>[데이터]<br><br>**해석 형식 예시:**<br>'ODA 변수의 계수 0.15는 ODA가 1% 증가할 때 취학률이 0.15%p 상승함을 의미하며, p-value 0.03으로 5% 유의수준에서 통계적으로 유의하다.'<br><br>위 형식을 참고하여 **단계별로** 분석해주세요:<br>1. 각 변수의 계수와 유의성 해석<br>2. 모델 전체의 설명력 평가<br>3. 잠재적 한계점 (내생성, 누락변수 등)<br>4. 정책적 함의" |

#### Module 5: 정책 문서 작성 지원받기

| 항목 | 내용 |
|------|------|
| **학습 목표** | 브리핑, 정책 메모, 제안서 등 실무 문서 작성에 AI 활용 |
| **내재된 테크닉** | Persona 설정 + Few-shot Learning + Constraints |
| **정책 맥락** | 브리핑/메모/제안서 (인턴십, 수업 과제) |

**소크라테스 질문 예시**:
- "정책 브리핑과 학술 논문의 문체는 어떻게 다른가요?"
- "독자가 누구인지 명시하면 AI 응답이 어떻게 달라질까요?"

**시나리오 A - 정책 브리핑**:
```
KDI에서 인턴 중입니다. 부장님이 "한국 저출산 대응 정책 현황"에 
대한 2페이지 브리핑 자료를 내일까지 요청하셨습니다.
AI에게 어떻게 초안 작성을 요청하시겠습니까?
```

**시나리오 B - 정책 메모**:
```
교수님께 연구 진행 상황을 보고하는 1페이지 메모를 작성해야 합니다.
연구 주제는 "디지털 전환이 공공서비스 전달에 미치는 영향"입니다.
```

**비교 실험 (시나리오 A)**:

| 학생 프롬프트 (예상) | 개선된 프롬프트 |

### 4.7 테크닉 통합 요약

| 테크닉 | 설명 | 적용 모듈 |

```
   │            │            │            │            │

                    점진적 테크닉 누적
```

| 위치 | 보여주는 정보 | 목적 |
|------|---------------|------|

#### 모듈 잠금 정책

| 정책 | 설명 |
|------|------|
| **순차 해제 (적용)** | 이전 모듈 완료 시 다음 모듈 해제 |

순차 해제 적용 이유:
- 테크닉이 점진적으로 누적되는 구조
- 학습 동기 부여 (다음 모듈 "해금" 효과)
- 기초 없이 고급 모듈 진입 방지

#### 진도 시각화 UI 컴포넌트

| **TechniquesBadge** | 획득 테크닉 | 완료/진행중/미획득 상태 |

### 5.4 모듈 학습 페이지 내부 구성 (스텝 통합 방식)

각 모듈 페이지(`/modules/1` ~ `/modules/5`)는 단일 페이지 내 스텝 전환 방식으로 구현

```
Module 페이지 내부 스텝 (탭 또는 스테퍼)
┌─────────────────────────────────────────────────────────┐
│  Step 1        Step 2        Step 3        Step 4      │
│  소크라테스 → 프롬프트 작성 → 비교 실험실 → 성찰 저널   │
│  대화                                                   │
└─────────────────────────────────────────────────────────┘
```

**스텝 통합 방식의 장점**:
- 학습 흐름의 연속성 유지
- 페이지 이동 시 컨텍스트 손실 방지
- 진행 상태 시각화 용이

```
로그인 → 학생 대시보드 → 모듈 선택 → 모듈 학습 페이지
                                         ↓
                              Step 1: 소크라테스 대화

                              Step 2: 프롬프트 작성

                                         ↓
                              Step 4: 성찰 저널
                              (배운 점 정리)
                                         ↓
                              모듈 완료 → 다음 모듈 해제
                                         ↓
                              대시보드 진도 업데이트
```

```
로그인 → 교수자 대시보드 → 학생 진도 현황 확인
```

```sql
profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),

  preferred_language TEXT DEFAULT 'ko',

  department TEXT
)

  id UUID PRIMARY KEY,
  title TEXT NOT NULL,
  title_en TEXT,

  prerequisite_module_id UUID REFERENCES modules(id)

scenarios (

  improved_prompt_example TEXT
)

)

-- 테크닉 배지
technique_badges (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id),
  technique_name TEXT NOT NULL,
  module_id UUID REFERENCES modules(id),

)
```

## 부록 B: 동료 평가 기능 (Phase 3)

### B.1 기능 설명

학생들이 서로의 프롬프트를 평가하고 피드백을 주고받는 기능

### B.2 작동 방식

```
1. 학생 A가 시나리오에 대한 프롬프트 제출
              ↓
2. 익명으로 다른 학생 2-3명에게 배정
              ↓
3. 평가자가 프롬프트를 보고:
   - 강점/약점 코멘트 작성
   - 항목별 점수 부여 (구체성, 맥락 등)
   - 개선 제안 작성
              ↓
4. 학생 A가 피드백 수령 + 자기 프롬프트 수정 기회
```

### B.3 평가 항목

| 항목 | 평가 질문 |
|------|-----------|
| 구체성 | 이 프롬프트가 충분히 구체적인가요? |
| 맥락 | 배경 정보가 적절히 제공되었나요? |
| 명확성 | AI가 무엇을 해야 하는지 명확한가요? |
| 개선점 | 어떻게 수정하면 더 좋을까요? |

### B.4 교육적 효과

- **평가하면서 배움**: 타인의 프롬프트 분석 → 자기 프롬프트 점검 능력 향상
- **다양한 접근법 노출**: 같은 시나리오에 대한 다양한 프롬프트 스타일 학습
- **협력 학습**: 동료 피드백이 AI 피드백보다 더 공감되는 경우 존재
